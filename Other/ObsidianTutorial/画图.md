## 系统设计
### 功能结构设计
```mermaid
graph TB
    A[心理量表测试系统] --> B[用户管理]
    A --> C[心理量表管理]
    A --> D[测评管理]
    A --> E[数据统计与分析]

    B --> B1[用户注册]
    B --> B2[用户登录]
    B --> B3[用户信息维护]

    C --> C1[量表创建]
    C --> C2[量表编辑]
    C --> C3[量表删除]
    C --> C4[量表查看]

    D --> D1[测评开始]
    D --> D2[测评提交]
    D --> D3[测评结果查看]

    E --> E2[统计分析报告]
    E --> E3[历史数据查询]

```
### 流程图设计
#### 注册流程
```mermaid
graph TD
    A[用户访问注册页面] --> B[输入用户名、密码和个人信息]
    B --> C[系统验证信息合法性]
    C --> D{用户名是否已存在？}
    D -- 是 --> E[提示用户名已存在，重新输入]
    D -- 否 --> F{密码格式是否正确？}
    F -- 否 --> G[提示密码格式错误，重新输入]
    F -- 是 --> H[提交注册信息]
    H --> I[系统存储用户信息到用户表中]
    I --> J[注册成功]

```
#### 量表测评流程
```mermaid
graph TD
    A[用户登录系统] --> B[选择心理量表进行测评]
    B --> C[系统展示心理量表的题目]
    C --> D[用户逐题作答]
    D --> E[用户完成测评]
    E --> F[系统保存填写记录到填写记录表中]
    F --> G[系统计算测评结果]
    G --> H[展示测评结果]

```
#### 统计分析流程
```mermaid
graph TD
    A[系统获取用户的填写记录] --> B[展示数据结果]
    B --> C[用户查看测评结果]

```
